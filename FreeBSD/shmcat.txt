NAME
       ftok - System V Interprozesskommunikationsschlüssel durch die
       Bibliotheksfunktion ftok(3) berechnen

ÜBERSICHT
       ftok [ OPTION ] DATEINAME ID

BESCHREIBUNG
       Einen System V Interprozesskommunikationsschlüssel aus dem angegebenen
       DATEINAMEn und der ID durch die Bibliotheksfunktion ftok(3) berechnen
       und das Ergebnis an die Standardausgabe übermitteln. Wurde nichts
       gegenteiliges angegeben, so wird der Schlüssel in hexadezimaler Form
       ausgegeben, wobei ihm das Präfix 0x voran gestellt wird.

       DATEINAME
              Der Name einer existierenden Datei. Dabei muss es sich nicht
              unbedingt um eine reguläre Datei handeln. Mit Verzeichnissen
              oder Spezialdateien sollte es auch funktionieren.

       ID     Eine Zahl zwischen 0 und 255, jeweils inklusive.

       Wenn shmcat ohne Unterstützung für getopt_long(3) kompiliert wurde,
       dann können nur die kurzen, aus einem Buchstaben bestehenden OPTIONen
       verwendet werden. Ansonsten werden auch die langen Varianten
       unterstützt.

OPTIONEN
       -d, --decimal
              Das Ergebnis in dezimaler Form, ohne Präfix ausgeben.

       -h, -?, --help
              Informationen zur Verwendung des Programms anzeigen und ohne
              weitere Aktionen beeenden.

       -o, --octal
              Das Ergebnis in oktaler Form mit einer führenden Null (0) als
              Präfix ausgeben.

       -V, --version
              Versionsinformationen anzeigen und beenden.

RÜCKGABEWERTE
       0      Kein Fehler, der Schlüssel konnte berechnet und ausgegeben
              werden.

       1      Beim Berechnen oder Ausgeben des Schlüssels ist ein Fehler
              aufgetreten.

       10     Das Programm wurde falsch aufgerufen.

PROBLEME
       Das Kommando ftok(1) hat die gleichen Beschränkungen wie die
       Bibliotheksfunktion ftok(3). Das bedeutet, dass es keine Garantie dafür
       gibt, dass der berechnete Schlüssel eindeutig ist.

       Unter Linux scheint es möglich zu sein, 0 als ID (zweiter Parameter) an
       die Funktion ftok(3) zu übergeben, aber dieses Verhalten ist nicht
       spezifiziert. Das Kommando ftok(1) akzeptiert 0 als ID und überlässt es
       der Bibliotheksimplementierung, was damit zu tun ist.
