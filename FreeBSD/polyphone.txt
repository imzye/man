NOM
       polyphone - éditeur de banques de sons


SYNOPSIS
       polyphone
       polyphone [FICHIER]
       polyphone [FICHIER_1] [FICHIER_2] ...
       polyphone -1 [-i CHEMIN_DE_FICHIER_EN_ENTRÉE] [-d RÉPERTOIRE_EN_SORTIE]
       [-o NOM_EN_SORTIE]
       polyphone -2 [-i CHEMIN_DE_FICHIER_EN_ENTRÉE] [-d RÉPERTOIRE_EN_SORTIE]
       [-o NOM_EN_SORTIE] [-c CONFIG]
       polyphone -3 [-i CHEMIN_DE_FICHIER_EN_ENTRÉE] [-d RÉPERTOIRE_EN_SORTIE]
       [-o NOM_EN_SORTIE] [-c CONFIG]
       polyphone -s [-i CHEMIN_DE_FICHIER_EN_ENTRÉE] [-c CONFIG]


DESCRIPTION
       polyphone fournit une interface simple et efficace pour créer et éditer
       des fichiers .sf2 . Ce logiciel incorpore des outils pour faciliter et
       automatiser l'édition de différents paramètres, rendant possible la
       gestion de grandes quantités de données.

       Les formats sf2, sf3, sfz (import / export), sfArk (import uniquement)
       et GrandOrgue (import uniquement) sont supportés. Les lignes de
       commande peuvent être utilisées pour convertir des fichiers, ouvrir
       Polyphone en mode synthétiseur ou réinitialiser les configurations en
       cas de soucis.

       Davantage d'informations peuvent être trouvées sur le site
       https://www.polyphone.io .


OPTIONS
       FICHIER
              Spécifie le fichier à ouvrir quand polyphone démarre. Plusieurs
              fichiers peuvent être chargés s'ils sont séparés par des
              espaces.  Les formats sf2 (version 2.01 ou 2.04), sf3, sfz,
              sfArk (version 1 ou 2) et GrandOrgue sont supportés.

       -1     Utiliser polyphone pour convertir un fichier dans le format sf2.

       -2     Utiliser polyphone pour convertir un fichier dans le format sf3.

       -3     Utiliser polyphone pour convertir un fichier dans le format sfz.

       -s     Ouvrir polyphone en mode synthétiseur.

       [-i CHEMIN_DU_FICHIER_EN_ENTRÉE]
              Chemin du fichier d'entrée à convertir ou ouvrir. Le format du
              fichier d'entrée doit être sf2, sf3, sfz, sfArk ou organ.

       [-d RÉPERTOIRE_DE_SORTIE]
              Le répertoire de sortie dans lequel le fichier d'entrée sera
              converti. Par défaut, le même répertoire que celui du fichier
              d'entrée est utilisé.

       [-o NOM_DE_SORTIE]
              Le nom de sortie du fichier converti. L'extension sera ajoutée
              automatiquement suivant la conversion. Par défaut, le même nom
              que le fichier d'entrée est utilisé.

       [-c CONFIG]
              Configuration de la conversion, le contenu étant dépendant du
              type de conversion.  -r Supprime la configuration existante.
               * conversion sf2
              Pas de configuration pour cette conversion.
               * conversion sf3
              La configuration est faite avec un caractère indiquant la
              qualité de la compression. '0' pour basse, '1' pour moyenne, et
              '2' pour haute qualité. Le défaut est de '1'.
               * conversion sfz
              La configuration est composée de 3 valeurs séparées d'un '|'. La
              première valeur est '1' si chaque ensemble doit être préfixé par
              son numéro d'ensemble, '0' sinon. La deuxième valeur est '1' si
              un répertoire par banque doit être créé, '0' sinon. La troisième
              valeur est '1' si la classification General MIDI doit être
              utilisée pour trier les ensembles, '0' sinon. Par défaut,
              '0|0|0' est utilisé.
               * mode synthétiseur
              La configuration est composée de 3 valeurs séparées d'un '|'. La
              première valeur désigne le canal MIDI à écouter, qui peut être
              une valeur de 1 à 16. La valeur 0 ou « all » indique que tous
              les canaux seront utilisés et seront traités comme un seul
              canal. La valeur -1 ou « each » écoute chaque canal séparément,
              ce qui constitue un prérequis pour lire des fichiers MIDI par
              exemple. La deuxième valeur est 'on' ou 'off' pour rendre
              possible la lecture de plusieurs ensembles en même temps. La
              troisième valeur est 'on', 'off' ou 'toggle' pour permettre
              l'activation ou la désactivation des ensembles par les notes les
              plus basses du clavier. Par défaut, 'all|off|off' est utilisé.

EXEMPLES
        * Conversion depuis sfArk vers sf2 :
       polyphone -1 -i /chemin/vers/le/fichier.sfArk
        * Conversion depuis sf2 vers sf3 (bonne qualité) :
       polyphone -2 -i /chemin/vers/le/fichier.sf2 -c 2
        * Conversion depuis sf3 vers sfz (sous-répertoire pour la banque et
       classification GM) :
       polyphone -3 -i /chemin/vers/le/fichier.sf3 -c '0|1|1'
        * Ouverture de Polyphone en mode synthétiseur en permettant l'usage
       des touches basse pour la sélection de l'ensemble à jouer avec un
       clavier MIDI :
       polyphone -s -i /chemin/vers/le/fichier.sf2 -c 'all|off|toggle'
