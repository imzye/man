NAME
       validxml - Command-line driver for XML::Validate.

SYNOPSIS
           validxml *.xml
           validxml --assert-invalid *.xml

DESCRIPTION
       Command-line driver for XML::Validate using the 'BestAvailable'
       processing type.

   Commandline Options
       --assert-invalid
           Swap the ok/not ok so invalid things are OK - still output the
           validation error) - this is useful for schema "unit tests".

       --backend [validator type]
           Specify an "XML::Validate" backend to use (e.g LibXML, Xerces).
           Defaults to BestAvailable.

   Output
       Output is Test::Harness compatible.

           1..scalar @files
           ok - filename/not ok - filename - validation error
