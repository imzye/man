NAME
       nbd-clid

DESCRIPTION
       nbd-clid is a client side daemon, which will handle the cmd requests from nbd-cli and do the stale connections restore when the node or the nbd-runner is reboot

SYNOPSIS
       nbd-clid [OPTION]

OPTIONS
       -r, --rhost=<RUNNER_HOST>
              Specify  the  host  where the nbd-runner service is running and listening for the RPC Control command requests, such as create/delete/map/unmap/list, etc from the nbd-clid/nbd-cli side, "local‚Äê
              host" as default.

       -u, --uid=<UID>
              Run as uid, default is current user

       -g, --gid=<GID>
              Run as gid, default is current user group

       -h, --help
              Display this help and exit

       -v, --version
              Display version and exit

       NOTE:  The RUNNER_HOST will be useful if the nbd-runner and nbd-clid services are working on diffrent nodes.
