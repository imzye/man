NAME
       gluster-blockd - Gluster Block Management Daemon

SYNOPSIS
       gluster-blockd [OPTION...]

DESCRIPTION
       gluster-blockd  daemon  is  responsible for block management, hence the
       daemon must be running on all servers.

OPTIONS
   Basic Options
       --glfs-lru-count <COUNT>
              glfs objects cache capacity [max: 512] [default: 5]

       --log-level <LOGLEVEL>
              Logging severity. Valid options are TRACE, DEBUG, INFO, WARNING,
              ERROR, CRIT and NONE [default: INFO].

       --no-remote-rpc
              Ignore  remote  rpc  communication, capabilities check and other
              node sanity checks

   Miscellaneous Options
        usage | help
              Display a usage message.

        version
              Print the program version.

EXAMPLES
       With lru cache capacity 10
       # gluster-blockd --glfs-lru-count 10

       To choose log-level DEBUG
       # gluster-blockd --log-level DEBUG

       With lru cache capacity 8 and log-level ERROR
       # gluster-blockd --glfs-lru-count 8 --log-level ERROR

       Run gluster-blockd in ignore remote rpc mode
       # gluster-blockd --no-remote-rpc

FILES

       /var/run/gluster-blockd.socket
