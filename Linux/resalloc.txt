NAME
       resalloc

SYNOPSIS
       resalloc [-h] [--connection CONNECTION] [--version] {ticket,ticket-check,ticket-wait,ticket-close} ...

OPTIONS
       --connection CONNECTION

       --version
              show program's version number and exit

   Sub-commands
       resalloc ticket
              Create ticket

       resalloc ticket-check
              Obtain ticket

       resalloc ticket-wait
              Wait till ticket is ready and write the output

       resalloc ticket-close
              Close a ticket

OPTIONS 'resalloc ticket'
       usage: resalloc ticket [-h] --tag TAGS [--sandbox SANDBOX]

       --tag TAGS
              What tag the Resource should have

       --sandbox SANDBOX
              Place the assigned resource into sandbox, and allow it's re-use for subsequent
              tickets (having the same --sandbox). This makes your ticket resolution faster,
              depending whether the server allows this feature or not. When --sandbox isn't
              specified, server uses randomly generated unique value and never re-uses the
              assigned resource.

OPTIONS 'resalloc ticket-check'
       usage: resalloc ticket-check [-h] ticket

       ticket Get the ticket

OPTIONS 'resalloc ticket-wait'
       usage: resalloc ticket-wait [-h] ticket

       ticket ID of ticket to wait for

OPTIONS 'resalloc ticket-close'
       usage: resalloc ticket-close [-h] ticket

       ticket ID of ticket to be closed
